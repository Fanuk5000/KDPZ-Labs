# Лабораторна робота 3
## Програмне забезпечання для тестування знянь

## Відповіді на контрольні питання

1. **Змінні, константи та літерали у C++. Область видимості.**
* **Змінні**: іменовані ділянки пам'яті.
* **Константи**: значення, що не змінюються після ініціалізації (визначаються ключовими словами `const` або `constexpr` для обчислення на етапі компіляції).
* **Літерали**: жорстко закодовані значення (наприклад, `42`, `3.14f`, `"Hello"`). Дозволені користувацькі літерали (user-defined literals).
* **Область видимості (Scope)**: локальна (в межах блоку `{}`), глобальна, область видимості класу та простору імен (namespace).


2. **Типи даних. Приведення типів.**
* **Типи**: Фундаментальні (`int`, `float`, `double`, `char`, `bool`), складені (масиви, вказівники, посилання) та користувацькі (класи, структури, перелічення `enum class`).
* **Приведення**:
* У стилі C: `(int)x` (не рекомендується).
* У стилі C++: `static_cast` (безпечне приведення під час компіляції), `dynamic_cast` (для поліморфних типів під час виконання), `const_cast` (зняття константності), `reinterpret_cast` (низькорівневе приведення байтів).




3. **Namespace у C++.**
* Замість пакетів (packages) у C++ використовуються **простори імен (`namespace`)**. Вони слугують для уникнення конфліктів імен. Наприклад, `std::cout` означає, що об'єкт `cout` належить до простору імен `std`.


4. **Інструкції та оператори.**
* **Інструкції (statements)**: найменша незалежна одиниця коду, зазвичай закінчується крапкою з комою (`;`).
* **Оператори**: арифметичні (`+`, `-`, `*`, `/`, `%`), логічні (`&&`, `||`, `!`), побітові (`&`, `|`, `^`, `<<`, `>>`), присвоювання (`=`, `+=`), порівняння (`==`, `!=`).


5. **Умови та цикли. Переривання.**
* **Умови**: `if`, `else if`, `else`, `switch` (у C++ switch працює лише з цілочисельними типами або `enum`).
* **Цикли**:
* Передумова: `while (condition)`.
* Післяумова: `do { ... } while (condition)`.
* З параметром: класичний `for (int i = 0; i < 10; ++i)` та range-based for: `for (auto& item : collection)`.


* **Переривання**: `break` (вихід з циклу), `continue` (перехід на наступну ітерацію).


6. **Масиви у C++.**
* **C-style масиви**: `int arr[10];` (небезпечні, не знають свого розміру при передачі у функції).
* **Стандартна бібліотека (STL)**:
* `std::array<int, 10>` — статичний масив фіксованого розміру.
* `std::vector<int>` — динамічний масив (аналог `List` або `ArrayList` з інших мов), який може змінювати розмір.




7. **Методи/функції. Параметри. Повернення значення.**
* Функції можуть бути глобальними або методами класу.
* **Передача параметрів**:
* За значенням (`void foo(int x)`): копіювання об'єкта.
* За вказівником (`void foo(int* x)`): передача адреси пам'яті.
* За посиланням (`void foo(int& x)` або `const int& x`): найефективніший спосіб передати великий об'єкт без копіювання.


* **Повернення**: за значенням, посиланням (небезпечно повертати локальні змінні) або вказівником.


8. **Процедурний vs ООП підхід.**
* **Процедурний**: програма ділиться на функції та структури даних окремо. Головний фокус на логіці (алгоритмах).
* **ООП**: дані та методи, які ними керують, об'єднуються в єдині сутності — об'єкти. Фокус на взаємодії об'єктів.


9. **Концепція ООП.**
* Це парадигма програмування, де архітектура програми будується як сукупність об'єктів, кожен з яких є екземпляром певного класу, а класи утворюють ієрархію успадкування.


10. **Принципи ООП (приклади C++).**
* **Абстракція**: Виділення головного. Клас `Car` містить `startEngine()`, приховуючи складні процеси подачі палива.
* **Інкапсуляція**: Приховування даних за допомогою модифікаторів доступу. Поле `private: int speed;` і доступ до нього через `public: void setSpeed(int s);`.
* **Спадкування**: Створення нового класу на основі існуючого. `class Dog : public Animal {};`.
* **Поліморфізм**: Можливість обробляти різні об'єкти через спільний інтерфейс. Використання ключового слова `virtual` у базовому класі та його перевизначення (`override`) у похідних.


11. **Що таке клас?**
* Клас — це шаблон (креслення), за яким створюються об'єкти.
* **Оголошення**: `class MyClass { public: int data; void doSomething(); };`


12. **Що таке об'єкт? Створення у C++.**
* Об'єкт — це конкретний екземпляр класу в пам'яті.
* На стеку (автоматичне видалення): `MyClass obj;`
* У купі (ручне управління): `MyClass* obj = new MyClass();` (Потребує `delete obj;`). Сучасний C++ вимагає використання розумних вказівників: `std::unique_ptr<MyClass> obj = std::make_unique<MyClass>();`.


13. **Конструктори та деструктори.**
* **Конструктор**: метод, що викликається при створенні об'єкта (не має типу повернення). Типи: за замовчуванням, з параметрами, конструктор копіювання (`MyClass(const MyClass&)`), конструктор переміщення (`MyClass(MyClass&&)`).
* **Деструктор**: метод для очищення ресурсів (`~MyClass()`). У C++ він відіграє критичну роль завдяки парадигмі RAII (Resource Acquisition Is Initialization).


14. **Модифікатори доступу.**
* `public`: доступно всім.
* `protected`: доступно класу та його спадкоємцям.
* `private`: доступно лише методам самого класу. Це прямий інструмент реалізації **інкапсуляції**.


15. **Класи, структури та записи.**
* У C++ `class` та `struct` практично ідентичні. Єдина відмінність: за замовчуванням у `class` всі члени та успадкування є `private`, а у `struct` — `public`.
* Окремого типу "запис" (record) у C++ немає, його роль виконують `struct` (Plain Old Data).


16. **Перевантаження операторів.**
* Дозволяє задати поведінку операторів (`+`, `==`, `<<`) для власних типів даних.
* **Обмеження**: не можна перевантажити `::` (область видимості), `.` (доступ до члена), `.*` (доступ за вказівником), `?:` (тернарний), `sizeof`, `typeid`. Також не можна створювати нові оператори (наприклад, ``).
* **Приклади**:
* Унарний: `MyClass& operator++();`
* Бінарний: `MyClass operator+(const MyClass& rhs) const;`




17. **Суть механізму успадкування.**
* Встановлення відношення "батько-нащадок". Дозволяє перевикористовувати код базового класу та розширювати його логіку, не дублюючи код.


18. **Що успадковується / не успадковується.**
* **Успадковуються**: всі поля і методи базового класу (доступ до `private` полів напряму неможливий, але вони фізично є в об'єкті).
* **Не успадковуються**: конструктори, деструктори, перевантажені оператори присвоювання (`operator=`), дружні функції (`friend`). *Примітка: у C++11 конструктори можна імпортувати через `using Base::Base;`.*


19. **Особливості виклику конструкторів при успадкуванні.**
* Спочатку викликається конструктор базового класу (через список ініціалізації нащадка), потім конструктор класу-нащадка.
* Деструктори викликаються у зворотному порядку: спочатку деструктор нащадка, потім базового класу (деструктор базового класу обов'язково має бути `virtual`).


20. **Одиночне та множинне успадкування.**
* C++ (на відміну від Java чи C#) **повністю підтримує множинне успадкування**: `class Child : public Parent1, public Parent2`.
* **Проблема**: "Ромбоподібне успадкування" (Diamond problem) — коли два базових класи успадковують один і той самий суперклас, нащадок отримує дві копії суперкласу.
* **Рішення**: Віртуальне успадкування (`class Parent1 : virtual public SuperClass`).


21. **Реалізація поліморфізму у C++.**
* Реалізується через механізм **таблиці віртуальних функцій (vtable)**. У базовому класі метод позначається словом `virtual`. Під час виконання програми (run-time) система перевіряє реальний тип об'єкта, на який вказує вказівник або посилання, і викликає відповідний метод.


22. **Абстрактні, статичні та запечатані класи.**
* **Абстрактні**: клас стає абстрактним, якщо в ньому є хоча б одна чисто віртуальна функція (pure virtual function), наприклад: `virtual void draw() = 0;`. Створити об'єкт такого класу неможливо.
* **Статичні**: у C++ немає статичних класів, але клас може містити виключно статичні поля і методи (`static`).
* **Запечатані (sealed)**: у C++ (починаючи з C++11) використовується ключове слово `final`. `class MyClass final {};` (від нього не можна успадковувати).


23. **Раннє та пізнє зв'язування.**
* **Раннє (статичне, compile-time)**: адреса функції відома на етапі компіляції (звичайні методи, перевантажені функції, шаблони).
* **Пізнє (динамічне, run-time)**: адреса визначається під час виконання програми через vtable (віртуальні функції).


24. **Відношення is-a, has-a.**
* **is-a (є чимось)**: реалізується через публічне успадкування. (Наприклад, `class Car : public Vehicle` -> Машина *є* транспортним засобом).
* **has-a (містить щось)**: композиція або агрегація. Клас містить об'єкт іншого класу як поле (Наприклад, `class Car { Engine eng; };` -> Машина *містить* двигун).


25. **Винятки у C++.**
* Робота через блоки `try`, `catch`, `throw`.
* **Вбудовані**: знаходяться в бібліотеці `<stdexcept>` (наприклад, `std::out_of_range`, `std::runtime_error`).
* **Користувальницькі**: створюються шляхом успадкування від `std::exception` та перевизначення методу `const char* what() const noexcept`. На відміну від Java, у C++ немає блоку `finally` (його замінює парадигма RAII).


26. **Індексатори та властивості.**
* У C++ немає вбудованих "властивостей" (properties) на рівні синтаксису. Замість них пишуть звичайні методи-геттери (getters) та сеттери (setters).
* **Індексатори** реалізуються виключно через перевантаження оператора доступу за індексом: `ReturnType& operator[](int index);`.


27. **Узагальнені типи.**
* У C++ це називається **Шаблони (Templates)**. `template <typename T> class Box { T item; };`.
* Шаблони C++ є Тюрінг-повними і набагато потужнішими за Generics у Java/C#. При компіляції для кожного використаного типу генерується окремий фізичний код (мономорфізація).


28. **Делегати.**
* Ключового слова `delegate` у C++ немає.
* Замість них використовують: вказівники на функції (C-style), функтори (об'єкти з перевантаженим `operator()`), **лямбда-вирази** (C++11) та обгортку `std::function` з бібліотеки `<functional>`.


29. **Інтерфейси у C++.**
* У C++ немає ключового слова `interface`.
* Інтерфейс створюється як абстрактний клас, який не має жодних полів (змінних-членів) і містить **виключно чисто віртуальні функції** (`= 0`) та віртуальний деструктор.


30. **Технології XML та JSON у C++.**
* Стандартна бібліотека C++ (STL) **не має** вбудованих засобів для парсингу XML чи JSON.
* Для роботи з ними використовують сторонні бібліотеки (third-party): `nlohmann/json` або `RapidJSON` для JSON; `pugixml` або `TinyXML` для XML.


31. **Робота з файловою системою.**
* Для читання/запису файлів використовуються файлові потоки з бібліотеки `<fstream>` (`std::ifstream` для читання, `std::ofstream` для запису).
* Для роботи безпосередньо з файловою системою (створення папок, перевірка існування файлу, обхід директорій) починаючи з C++17 використовується потужна бібліотека **`<filesystem>`** (простір імен `std::filesystem`).
